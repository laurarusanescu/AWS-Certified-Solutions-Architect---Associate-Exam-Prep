# Architecting Solutions on AWS — Course Overview Notes

---

## Course Context
- **Course name**: Architecting Solutions on AWS  
- **Program**: AWS Cloud Solutions Architect Professional Certificate  
- **Position**: 2nd course out of 4

---

## Course Goal (EXAM & CAREER FOCUSED)
This course prepares you to **design real-world AWS architectures** by:
- Analyzing business requirements
- Selecting appropriate AWS services
- Designing solutions that are:
  - Resilient
  - Secure
  - Highly available
  - Cost-efficient

Learning is driven by **use case–based scenarios**.

---

## Learning Approach
- Each week introduces a **fictional customer**
- Customer presents:
  - Business needs
  - Constraints
  - Problems to solve
- Instructors:
  - Evaluate architectural options
  - Select best-fit AWS services
  - Walk through the design on a whiteboard

---

## Key Architecture Themes
- Event-driven architectures
- Serverless design
- Purpose-built AWS services
- Hybrid architectures
- Reliability & operational efficiency
- Security-focused multi-account strategies
- Cost optimization

---

## Course Structure
- Divided into **weekly modules**
- Self-paced learning
- Weekly structure designed to complete the certificate in ~**6 months**

---

## Week 1: Designing a Serverless Web Backend
**Use case**: Ecommerce company migrating a backend service to AWS

### Focus Areas
- Serverless architecture
- Performance efficiency
- Cost optimization

### Key AWS Services
- Amazon API Gateway
- AWS Lambda
- Amazon SQS
- Amazon SNS
- Amazon DynamoDB

---

## Week 2: Designing a Serverless Data Analytics Solution
**Use case**: Software company analyzing clickstream data

### Focus Areas
- Data ingestion
- Serverless analytics
- Insight generation

### Key AWS Services
- Amazon S3
- Amazon Kinesis Data Firehose
- Amazon Athena
- Amazon QuickSight
- Additional analytics services

---

## Week 3: Designing a Hybrid Container-Based Solution
**Use case**: Enterprise insurance company migrating container workloads using a hybrid model

### Focus Areas
- Hybrid architecture
- Containers
- Reliability

### Key AWS Services
- AWS Direct Connect
- Amazon ECS
- Amazon ECS Anywhere
- Amazon RDS
- AWS Systems Manager

---

## Week 4: Designing an Account Governance & Management Strategy
**Use case**: Startup defining account governance best practices

### Focus Areas
- Security
- Governance
- Cost management
- Standards enforcement

### Key AWS Services
- AWS Organizations
- AWS IAM Identity Center (successor to AWS SSO)
- AWS CloudTrail
- Supporting governance services

---

## Skills You Gain by the End of the Course
- Translate business requirements into AWS architectures
- Choose purpose-built AWS services confidently
- Design serverless and hybrid solutions
- Apply governance and security best practices
- Think like a Solutions Architect

---

## Exam & Career Takeaway Summary
- Architecture decisions are **use case driven**
- Serverless and event-driven patterns are core
- Hybrid designs are common in enterprises
- Governance and security are foundational
- Multi-account strategies are best practice
